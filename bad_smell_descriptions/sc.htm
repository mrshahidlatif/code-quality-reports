Spaghetti Code appears as a program or system that contains very little software structure. If developed using an object-oriented language, the software may include a small number of objects that contain methods with very large implementations that invoke a single, multistage process flow.

In the scatterplot, spaghetti code is represented in red dot
. The identification of spaghetti code comes from high values of respective metric (average method complexity). Blue dot represents class
free of spaghetti code .

A couple of steps can be taken to remove spaghetti code. First, gain abstract access to member variables of a class using accessor functions, and then convert a code segment into a function.
